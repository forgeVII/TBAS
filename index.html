01

System Overview

TBAS detects sound events using two electret microphones spaced 28 cm apart. By measuring the Time Difference of Arrival (TDOA) between the two microphones, the system calculates the angle of the incoming sound using physics of wave propagation. A servo motor physically rotates toward the detected angle. A TSOP1838 IR receiver mounted on the servo then performs an IFF (Identification Friend or Foe) check using a dynamic rolling code protocol.

All data is streamed in real time to a browser-based web interface over USB serial. The interface displays a live radar, IFF result, bearing angle, and event log. No additional software, drivers, or server is required — only Chrome or Edge.

Signal Flow

// TBAS signal chain Sound Event → Electret Mics (x2, 28cm apart) → LM393 Dual Comparator → Arduino Hardware Interrupts (D2, D3) → TDOA Calculation → asin() angle formula → Servo Rotates to Angle (D12) → TSOP1838 IR Receiver checks IFF (D4) → IR LED sends rolling code response (D5) → RGB LED local indicator (A3/A4/A5) → Serial Output → Web Interface (Chrome)
02

Operating Modes

TBAS operates in four modes. Switch between them by holding both buttons simultaneously for more than 500ms — this cycles through modes in order.

MANUAL
Servo controlled directly by buttons. IFF check runs on button release.
SURVEILLANCE
Servo sweeps to random positions. IFF checked twice at each stop.
SERVO TEST
TDOA values and angles printed to serial. Used during calibration.
ACOUSTIC
Waits for sound, calculates angle, points servo, runs IFF. 5s lockout after detection.

TDOA Formula

Sound direction is calculated using the inverse sine of the time difference:

angle = asin( 343 × Δt / 0.28 ) × (180 / π) + 90°

Where Δt is the time difference in seconds between the two microphone triggers. The result maps to a servo angle between 25° and 155°, with 90° as dead center. Any TDOA value exceeding 816 µs (the physical maximum at 28 cm spacing) is discarded as a reflection or noise event.

04

IFF System

IFF uses NEC protocol IR communication at 38 kHz. Instead of a fixed static code which could be intercepted and replayed, TBAS uses a dynamic rolling code.

code = ( KEY × counter + 5 ) mod 256

Both units share the same KEY (7) and maintain a synchronized counter. The counter increments on every successful Friend exchange, so the code changes every cycle. An intercepted code cannot be replayed because the expected code has already advanced.

// SECURITY NOTE Enemy TX mode deliberately sends an incorrect code — used to simulate an adversary and verify the system correctly classifies unknown IR sources as ENEMY.

Web Interface

The TBAS web interface connects directly to the Arduino over USB using the Web Serial API — no drivers, no server, no installation. Open in Chrome or Edge, click Connect, select the Arduino COM port, and the interface is live.

Key Wiring Notes

// MIC CIRCUIT — IMPORTANT Mic (+) connects to IN+ (Pin 3) of LM393. Pot wiper connects to IN− (Pin 2). This is opposite to typical comparator tutorials. Interrupt must be RISING not FALLING. One 10kΩ bias resistor from 5V to Mic(+). One 10kΩ pullup from OUT to 5V. Two independent pots share a common voltage divider (10kΩ–10kΩ from 5V to GND).
// POWER NOTE Servo draws significant current on movement. Use a 100µF electrolytic capacitor across servo power pins. Power from laptop USB is sufficient for lab use. For field deployment, use a power bank — do not use phone OTG directly with servo.
// TSOP1838 PINOUT With flat face toward you, pins left to right: OUT — GND — VCC. No resistor needed on signal pin. 100nF decoupling cap recommended between VCC and GND.
